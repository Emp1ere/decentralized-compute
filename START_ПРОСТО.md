# Запуск для обывателя (всё через интерфейс)

## Шаг 1. Запустить Docker Desktop

Включите **Docker Desktop** и дождитесь, пока он полностью загрузится.

## Шаг 2. Запустить систему одним кликом

Дважды нажмите на файл **`start.bat`** в папке проекта.

Скрипт сам:
- соберёт и запустит контейнеры;
- через несколько секунд откроет в браузере интерфейс по адресу http://localhost:8080.

## Шаг 3. Запуск воркера из интерфейса

1. В открывшейся вкладке нажмите **«Зарегистрироваться»** (или введите уже имеющийся API-ключ).
2. Перейдите на вкладку **«Работа»** или **«Контракты»**.
3. Нажмите **«Получить задачу»** или у нужного контракта — **«Взять задачу»**.
4. Нажмите **«Запустить воркер автоматически»**.

Воркер запустится сам, выполнит выбранную задачу и завершится. Никаких команд в терминале вводить не нужно.

---

**Остановка:** закройте окно с логами или в папке проекта выполните `docker-compose down`.

**Если кнопка «Запустить воркер» пишет ошибку:** убедитесь, что перед запуском `start.bat` был запущен Docker Desktop и что вы дождались полной загрузки контейнеров (около 10 секунд после появления «Запуск системы»).

---

### Если start.bat не запускается или окно сразу закрывается

1. Запустите **Docker Desktop** и дождитесь, пока он полностью загрузится.
2. Откройте **командную строку** (Win+R → `cmd` → Enter).
3. Перейдите в папку проекта, например:
   ```bat
   cd /d "C:\Users\ВАШЕ_ИМЯ\OneDrive\Рабочий стол\Прогер\distributed-compute"
   ```
   (подставьте свой путь к папке, где лежит `start.bat` и `docker-compose.yml`).
4. Выполните:
   ```bat
   start.bat
   ```
   или сразу:
   ```bat
   docker-compose up -d --build
   ```
5. Через 10–20 секунд откройте в браузере: http://localhost:8080

Так вы увидите сообщения об ошибках (например, «Docker не найден» или «файл не найден»), если что-то пойдёт не так.
