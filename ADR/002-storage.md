# ADR 002: MinIO вместо IPFS для хранилища

**Статус:** Принят  
**Дата:** 2026-02  
**Контекст:** DSCM v2, ТЗ раздел 2

## Решение

Использовать **MinIO** (S3-совместимое хранилище) вместо IPFS для артефактов вычислений.

- **Входные данные:** загрузка через S3 API
- **Результаты:** сохранение в бакеты с access control
- **Шифрование:** server-side encryption (SSE)
- **Регионы:** бакеты по регионам (RU/EU/US/CN)

## Обоснование

1. **Фиат-фокус:** MinIO не требует крипто-инфраструктуры
2. **Контроль доступа:** IAM, bucket policies, проще compliance
3. **Производительность:** предсказуемая latency, CDN-интеграция
4. **Совместимость:** S3 API — стандарт, легко мигрировать на AWS S3

## Последствия

- Новая зависимость: MinIO (Docker/self-hosted или облачный)
- Адаптер в Orchestrator для S3-операций
- Текущее локальное хранилище (data/ingestion, data/results) — временное; миграция в Phase 3
