# Интерфейс по принципам BOINC

## Анализ интерфейса BOINC (по официальной документации)

**Папка в системе:** в `C:\ProgramData\...\BOINC` лежит только ярлык «BOINC Manager.lnk» — сам интерфейс это десктоп-приложение, его нельзя «открыть как файл». Ниже — структура, собранная по [Simple view](https://boinc.berkeley.edu/wiki/Simple_view) и [Advanced view](https://boinc.berkeley.edu/wiki/Advanced_view).

### Два режима
- **Simple View** — по умолчанию: один экран, основные кнопки, выбор проекта/задачи, прогресс, «Work done» (кредиты).
- **Advanced View** — вкладки для детального мониторинга и управления.

### Advanced View — вкладки
| Вкладка BOINC | Назначение | Аналог в нашей системе |
|---------------|------------|-------------------------|
| **Notices** | Новости проектов, уведомления | Обзор / сводка узла |
| **Projects** | Список проектов, статус, команды (Update, Suspend, No new tasks…) | **Контракты** — список контрактов (sc-001, sc-002), объём работы, награда |
| **Tasks** | Список задач, статус (Running, Ready to report…), команды | **Задачи** — последние сданные работы (work_receipt из цепочки) |
| **Transfers** | Загрузки/выгрузки файлов | Не нужен (нет файлов); можно заменить на **Синхронизация** (блоки между узлами) или не делать |
| **Statistics** | Графики кредитов (user/host, по проектам) | **Статистика** — метрики узла, длина цепочки, баланс, счётчики запросов |
| **Disk** | Использование диска | Не нужен; вместо этого — **Блокчейн** (последние блоки и транзакции) |
| **Event Log** | Окно с логом событий/ошибок | Опционально: последние действия или заглушка |

### Принципы, которые переносим
1. **Два уровня:** простой вид (одна страница — статус, баланс, регистрация) и расширенный (вкладки).
2. **Вкладки сверху** для переключения разделов.
3. **Таблицы с колонками** (задачи, контракты, блоки) — по возможности с читаемыми заголовками.
4. **Кнопки команд** под списком (как в BOINC: Update, Suspend и т.д.) — у нас: «Обновить», «Зарегистрироваться», переключение узла.
5. **Один «текущий узел»** — как «Select Computer» в BOINC: выбор узла (этот сервер / другой URL) для просмотра данных.

## Реализованный вариант

- **Веб-дашборд** (одна страница), открывается по адресу узла: `http://localhost:5000/` или `http://localhost:5000/dashboard`.
- **Вкладки:** Обзор | Работа | Контракты | Задачи | Блокчейн | Статистика.
- **Обзор** — состояние узла (health), метрики (длина цепочки, вычислители, транзакции в очереди), блок «Вычислители»: текущий вычислитель, список с балансом/сдано работ, кнопки Использовать/Обновить/Удалить, добавление через Вход/Регистрацию или API-ключ. Текст согласован с логикой: награды на баланс выбранного вычислителя, воркер — с его ключом.
- **Работа** — получение задачи, выполнение в браузере или запуск воркера; пояснено, что награда идёт на баланс вычислителя, чей ключ используется.
- **Контракты** — таблица контрактов: название, категория, описание, ед. на сдачу, выполнено %, активные вычислители, осталось (ед. работы), награда за сдачу, кнопка «Взять задачу».
- **Задачи** — сданные работы (work_receipt) из блокчейна: блок, вычислитель, контракт, ед. работы.
- **Блокчейн** — последние блоки (индекс, хеш, кол-во транзакций).
- **Статистика** — длина цепочки, вычислителей (зарегистрировано), транзакций в очереди, счётчики запросов и ошибок.
- Переключение узла: ссылка «Обновить» и отображение текущего узла (URL).

Интерфейс приведён в соответствие с логикой проекта: вычислители, баланс и сданные работы в блокчейне, контракты с remaining_volume и наградой за сдачу. API: **GET /contracts**, **GET /chain**, **GET /metrics** и др.
